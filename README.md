# Linux & Docker Security Hardening Project

## Project Overview
This project demonstrates a two-phase security audit focused on identifying and mitigating data exposure vulnerabilities. I transitioned a vulnerable "default" environment into a hardened infrastructure using industry-standard security principles.

## Key Skills Demonstrated
* **System Hardening:** Implementing the Principle of Least Privilege (PoLP) via Linux ACLs (`chmod`).
* **Container Security:** Transitioning from insecure Docker environment variables to **Docker Secrets** and secure volume mounts.
* **Vulnerability Assessment:** Identifying information leakage and directory traversal risks.
* **Documentation:** Professional technical reporting for stakeholders.

## Environment
* **OS:** Ubuntu Linux (VMware)
* **Containerization:** Docker Desktop
* **Certifications Applied:** CompTIA Security+, ISC2 CC

## Lab Phases
1. **Linux Filesystem:** Fixed directory listing vulnerabilities and secured sensitive `.php` config files.
2. **Docker Security:** Identified plaintext passwords in container metadata and migrated to secure runtime secrets.
